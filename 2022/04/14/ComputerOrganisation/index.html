第二章 计算机系统组成

2.1 处理器

​ 处理器的功能是从主存储器中逐条取出指令、分析指令、执行指令的过程来执行计算机程序。

​ CPU有几个相对独立的部分组成:

​ 控制器：负责从主存储器中取出指令，分析指令的类型。

​ 算术逻辑部件：完成加法，逻辑与等算数逻辑运算来执行指令。

2.1.1 CPU组成

​ 下图中所有的组成部件叫做数据通路，包括寄存器、算术逻辑部件、和连接它们的总线。

​ 数据通路周期：ALU 将两个操作数进行运算，并将结果写回的过程（数据通路周期越短计算机也就越快）。

2.1.2 指令执行

​ 一般来说，指令执行大概分为以下步骤：

​ （1）从主存中取出一条指令存放到寄存器中。

​ （2）让程序计数器指向后面一条指令。

​ （3）判断指令的类型。

​ （4）若该指令用到某主存单元，将该单元中的数据存放到寄存器中。

​ （5）执行该指令。

​ （6）返回到第一步。

​ 这个过程通常称为：取出——译码——执行 周期，是所有计算机操作的核心。

2.1.3 RISC 和 CISC

​ 1980 年，由Berkeley 的一个由 David Patterson 和 Carlo 领导的研究小组开始设计不用解释器的超大规模集成电路 CPU 芯片。

​ 设计出能快速启动的指令是提高性能的关键，每秒钟启动指令的条数比一条指令的执行时间更加重要。

​ RISC 是精简指令计算机（Reduced Instruction Set Computer）。

​ CISC 是复杂指令计算机（Complex Instruction Set Computer）。

​ CISC 在某些方面也采用了 RISC 的思想。

2.1.4 现代计算机的设计原则

​ （1）所有指令由硬件直接执行。

​ （2）最大限度提高指令启动速度。

​ （3）指令应容易译码：指令规整，固定指令长度，指令中的字段数少一点。

​ （4）只允许读写主存指令访问主存。

​ （5）提供足够的寄存器。

1. 指令流水

​ 指令流水是指将一条指令的执行过程分解为更多的部分，每个部分分别由精心设计的硬件分别执行，都可以并行运行。

2.1.6 处理器级并行

​ 难以去除 CPU 产生的热量，是近10年CPU速度停滞不前的主要原因。

1. 数据并行计算机

​ 单指令流多数据流处理器, 或称 SIMD（Simple Instruction Mutiple Data-Stream）处理器由许多在不同数据集合上执行完全相同指令序列的处理器组成。

​ 现代的图形处理器（GPU）严重依赖 SIMD 处理器使用较少的晶体管来获得较高的处理能力。这是因为图形处理算法是高度规则的。

​ 向量处理器和 SIMD 处理器类似，都是在不同的数据对象上执行相同的操作。
